# 优化迭代循环 (Optimization Iteration)

当步骤标记为 `type: "optimize"` 时启动。

**快速评估**：基线指标已达标或无提升空间则跳过，记录原因。

## 完整循环

```
Round 1: [指标搜索] → [基线实现] → [基线评测] → [瓶颈分析] → [优化] → [评测对比]
  if 提升 < 20% 或用户满意 → 结束 | else → Round 2 (最多 3 轮)
```

## 指标搜索

WebSearch 不可用时使用内置模板：

| 领域 | 推荐指标 | 评测方式 |
|------|---------|---------|
| CSV/JSON 解析 | 吞吐量 (rows/s), 内存峰值 (MB) | time.perf_counter + tracemalloc |
| 模型训练 | Accuracy, F1, 训练时间, 推理延迟 | sklearn.metrics + timeit |
| 排序/搜索 | 耗时, OPS, P50/P99 延迟 | timeit, 多轮取平均 |
| IO 密集型 | 读写吞吐 (MB/s), IOPS | time + os 统计 |

指标须可量化、可复现、有公认度。

## 评测规范

- 硬件/数据集/指标/运行次数（至少 3 次取平均）统一定义
- 优先使用 pyperf 以减少系统噪声
- 计时与内存测量分开进行（tracemalloc 拖慢计时）
- 测量前 warmup 至少 3-5 次

## 终止条件

| 条件 | 行为 |
|------|------|
| 提升 > 50% | 结束 |
| 提升 20-50% | 可继续一轮 |
| 提升 < 20% 或连续两轮 < 5% | 已达边际效益，强制结束 |
| 超时（默认 15 分/轮） | 强制结束 |
| 用户说"够了" | 立即结束 |

## 瓶颈分析

cProfile / py-spy / tracemalloc / 大 O 推算 / 磁盘吞吐分析，至少做一种后再优化。
